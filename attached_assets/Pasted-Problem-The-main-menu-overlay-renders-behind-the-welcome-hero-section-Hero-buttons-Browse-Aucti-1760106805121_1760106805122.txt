Problem

The main menu/overlay renders behind the welcome/hero section.

Hero buttons (“Browse Auctions”, “Start Bidding Today”) stay clickable over the menu.

Menu background is too transparent.

Goal (acceptance criteria)

When the menu is open, it visually sits above everything on the page.

Only menu items are clickable; hero buttons are not.

Menu backdrop is slightly see-through (about 70–85% opacity), not fully transparent.

Works on desktop and mobile.

Implementation (CSS/JS)

Use whichever fits the stack (raw CSS or Tailwind).

CSS (vanilla)
/* Menu container */
.site-menu {
  position: fixed;     /* detach from page flow */
  inset: 0 auto auto 0;
  width: 100%;
  z-index: 1000;       /* above hero */
}

/* Backdrop behind the menu */
.site-menu__backdrop {
  position: fixed;
  inset: 0;
  background: rgba(0,0,0,0.25); /* ~75% page visible => tweak 0.25–0.35 */
  backdrop-filter: blur(2px);    /* optional */
  z-index: 999;                  /* just under menu panel */
}

/* Ensure hero never floats above */
.hero, .welcome, .landing-hero {
  position: relative;
  z-index: 0;
}

/* When menu is open, prevent clicks to page */
body.menu-open .page-root,
body.menu-open .hero,
body.menu-open main {
  pointer-events: none;
}
body.menu-open .site-menu,
body.menu-open .site-menu * {
  pointer-events: auto; /* keep menu clickable */
}

Tailwind (example classes)

Menu wrapper: fixed inset-0 z-[1000]

Backdrop: fixed inset-0 z-[999] bg-black/30 backdrop-blur-sm

Hero wrapper: relative z-0

When menu opens, add menu-open to <body> and toggle a utility like:

Page containers: pointer-events-none

Menu container: pointer-events-auto

JS (toggle)
function openMenu() {
  document.body.classList.add('menu-open');
  // show .site-menu & .site-menu__backdrop
}

function closeMenu() {
  document.body.classList.remove('menu-open');
  // hide .site-menu & .site-menu__backdrop
}

Builder-specific notes (if using a visual builder)

Set the menu layer to “Fixed/Sticky” at top and highest z-index (e.g., 1000+).

Ensure the hero (welcome section) has no higher z-index and no CSS transform/filter on a parent that creates a new stacking context (e.g., transform, filter, opacity, position: relative with z-index). If present, remove or move the menu outside that container.

Add a semi-transparent backdrop block under the menu and above the page content.

On “Menu Open” state, enable “Disable pointer events” (or add pointer-events: none) on the main page wrapper.

Quick debug checklist

 Menu wrapper is position: fixed with z-index >= 1000.

 No parent of the hero has transform/filter/opacity < 1/mix-blend-mode creating a stacking context above the menu.

 Backdrop opacity ≈ bg-black/30 (or rgba(0,0,0,0.3)).

 pointer-events: none applied to hero/page while menu is open.
